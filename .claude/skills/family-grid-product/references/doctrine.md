# Doctrine

## Thesis
Use modular structure to explain value fast. Make information easy to scan without flattening all priorities into equal cards.

## Visual grammar
- grouped modules
- clear proof and CTA placement
- density with order
- tile sizing that encodes importance
- practical polish over spectacle

## Typography
- heading hierarchy should speed up scanning
- subheads explain utility fast
- proof and metrics need compact clarity

## Components
- cards should vary by role, not only by content
- pricing, proof, and feature grids should show priority
- dashboards and explainers need strong grouping logic

## Motion
Keep motion restrained-to-supportive. Favor feedback and guided emphasis over cinematic sequences.

## Anti-patterns
- equal cards everywhere
- too many feature blocks above the fold
- gradients hiding weak hierarchy
- dashboard density with weak grouping
- proof disconnected from CTA
